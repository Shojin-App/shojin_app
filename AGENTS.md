# リポジトリ作業ガイド

## プロジェクト概要

- Android、iOS、デスクトップ、Webを対象とするFlutter/Dartアプリケーションです。
- `mise.toml`で固定されたFlutterバージョンを使用してください。FVMは使用しません。
- 開発先は`dev`ブランチです。作業ツリーにユーザーの未コミット変更がある場合、依頼と無関係な変更を編集しないでください。
- ユーザー向け文言は主に日本語です。既存UIと用語を統一してください。

## ディレクトリ構成

- `lib/main.dart`: アプリの初期化、5タブ構成、タブ保持アニメーション、半透明ボトムナビゲーション
- `lib/screens/`: 画面単位のUIと状態管理
- `lib/widgets/`: Monacoエディタ連携や共通の半透明AppBarを含む再利用可能なウィジェット
- `lib/providers/`: Providerを通じて公開するアプリ全体の設定と状態
- `lib/services/`: 通信、永続化、通知、コンテスト、コード実行、更新処理
- `test/`: ウィジェットテストとユニットテスト
- `third_party/`: リポジトリ内に配置した依存ライブラリ。通常の解析対象外であり、依頼で明示されない限り変更しないでください。

## UI実装上の制約

- メインのScaffoldは`extendBody: true`を使用し、ブラウザと問題画面を半透明ボトムナビゲーションの背後まで描画します。
- `NavigationBarM3E`はmediumサイズの高さ80論理ピクセルを使用し、端末下部のSafeAreaも追加します。最後まで読める必要があるスクロール領域には、この両方を考慮した末尾余白を確保してください。画面自体の描画をボトムナビゲーションより上で止めないでください。
- ホーム、問題、設定画面は、それぞれ上部の半透明AppBarを管理します。ブラウザとM3EエディタのAppBarは、メイン画面から上部SafeAreaを受け取ります。上部SafeAreaを二重に適用しないでください。
- AppBarとボトムナビゲーションの透明度は、どちらも`ThemeProvider.navBarOpacity`に連動させてください。
- 保持中の非選択タブは`ExcludeFocus`内に置いてください。これを外すと、画面遷移から戻った際に以前のTextFieldがソフトウェアキーボードを再表示する可能性があります。
- 標準コードエディタのガターは行番号専用です。追加のガター列を有効にする場合は幅も広げ、狭いスマートフォン画面で複数桁の行番号が表示できることを確認してください。

## 編集方針

- 変更したDartファイルには`dart format`を実行してください。
- `analysis_options.yaml`および既存のProvider・Materialの実装パターンに従ってください。
- 実装の意図は、可能な限りコード内のコメントとして残してください。特に、非自明な制約、定数や回避策を採用した理由、画面・ウィジェット間の相互作用、将来のリファクタリングで誤って壊しやすい挙動を記録してください。
- コメントには直後の処理内容を言い換えるだけでなく、なぜその構造が必要なのか、どの不変条件を守っているのかを書いてください。実装の挙動を変更した場合は、関連するコメントも更新または削除してください。
- 画面余白、AppBar、ナビゲーション、エディタのサイズを変更する場合は、エッジツーエッジ表示を維持し、狭いスマートフォン画面でも確認してください。
- `.dart_tool/`、`build/`、ローカライズ生成物、ライセンス生成物などの自動生成ファイルは、依頼に再生成が含まれない限り手動編集しないでください。

## 検証

最低限、次のコマンドを実行してください。

```powershell
dart format <変更したDartファイル>
flutter analyze --no-pub
flutter test --no-pub
git diff --check
```

対象を絞った変更では、まず関連テストを実行してください。引き渡し前には、可能な限り全テストを実行してください。
