# UI・機能改善計画

## 実装済み

- エディタ・設定画面にプログラミング言語アイコンを表示
- ビルド番号5回タップで開発者モードを有効化し、TeXテストを開発者向け項目へ移動
- ホームに次回ABCの時刻・リマインダー状態・設定導線を表示
- 問題未選択時はエディタのサンプル・提出ボタンを非表示
- 問題URLをクリップボードから貼り付け、または1タップでクリア
- コンテンツがボトムナビゲーションバー背面へ描画されないよう修正
- ホームウィジェットの並べ替え・表示切替を永続化
- おすすめ問題に問題タイトルを表示
- ブラウザの既定サイトに `atcoder.jp` を追加
- コンテスト通知を起動時・設定変更時に再予約
- HackGen通常版を削除し、HackGen35のみ同梱
- release-pleaseのバージョン追跡値をpubspecと同期
- `robots.txt` をWebビルドへ追加

## 既存対応を確認済み

- レート制限: `RequestRateLimiter` とテストが存在
- 問題文の出典・表示権利: 元ページ導線と `docs/LEGAL.md` が存在
- CHANGELOG自動化: release-please設定が存在
- 現在レート: AtCoderプロフィール解析を使ってホームのおすすめ欄に表示

## 次の実装候補

### 優先度: 高

- AC時アニメーション
  - サンプルテスト全件成功時の演出と、提出結果取得後の演出を別々に設計する
  - 低スペック端末とReduce Motion設定を考慮する
- ホームのレート統計・グラフ
  - AtCoderプロフィールと履歴HTMLから取得可能な範囲を調査する
  - キャッシュ、取得失敗時表示、アクセス頻度制限を先に定義する
- 通知の実機検証
  - Android 13以降、iOS、端末再起動後、タイムゾーン変更時を確認する

### 優先度: 中

- 実行・サンプル・履歴のFAB化
  - 3操作を単一FABへ詰めると発見性が落ちるため、画面幅が狭い場合だけSpeed Dial化する案を比較する
- 全体デザイン刷新
  - 色、余白、角丸、タイポグラフィ、カード階層をデザイントークン化する
  - ホーム、問題、エディタ、設定の順に画面単位で刷新する
- CHANGELOG表示の自動化
  - リリースノート生成だけでなく、アプリ内表示が必要ならGitHub Releases APIとの同期を追加する

### 要設計・要確認

- 限定公開コンテスト
  - WebViewの認証CookieをHTTP取得処理へ渡す場合、Cookieの保存範囲・暗号化・ログ出力禁止を設計する
  - AtCoderの利用規約とアクセス方針を確認してから実装する
- 問題文などの著作権・表示権利
  - 現状の出典表示と元ページ導線で十分か、AtCoder側へ確認する
  - 許諾条件が不明な間はキャッシュ期間と再配布範囲を最小化する
